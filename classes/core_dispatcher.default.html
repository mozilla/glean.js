shutdown

  • shutdown(): Promise<void>

  • Shutsdown the dispatcher.

    1. Executes all tasks launched prior to this one.
    2. Clears the queue of any tasks launched after this one (even persistent tasks).
    3. Puts the dispatcher in the Shutdown state.

    Note

    • This is a command like any other, if the dispatcher is uninitialized it will get executed when the dispatcher is initialized.
    • If the dispatcher is stopped, it is resumed and all pending tasks are executed.

    Returns Promise<void>

    A promise which resolves once shutdown is complete.

    • Defined in src/core/dispatcher.ts:401

stop

  • stop(priorityTask?): void

  • Enqueues a Stop command at the front of the queue and triggers execution.

    The Stop command will stop execution of current tasks and put the Dispatcher in a Stopped state.

    While stopped the dispatcher will still enqueue tasks but won't execute them.

    In order to re-start the dispatcher, call the resume method.

    Note

    In case a Shutdown command has been launched before this command, this command will result in the queue being cleared.

    Parameters

    • priorityTask: boolean = true

      Whether or not to launch the clear command as a priority task. This is true by default.

    Returns void

    • Defined in src/core/dispatcher.ts:366

testLaunch

  • testLaunch(task): Promise<void>

  • Launches a task in test mode.

    Note

    This function will resume the execution of tasks if the dispatcher was stopped and return the dispatcher back to an idle state.

    This is important in order not to hang forever in case the dispatcher is stopped.

    Errors

    This function will reject in case the task is not launched. Make sure the dispatcher is initialized or is not shutdown in these cases.

    Parameters

    • task: Task

      The task to launch.

    Returns Promise<void>

    A promise which only resolves once the task is done being executed or is guaranteed to not be executed ever i.e. if the queue gets cleared.
